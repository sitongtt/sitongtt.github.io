JSON

什么是JSON？

  1. 目前主流趋势：前后端分离、微服务；后来发现，可以把较重的服务端和较轻的前端分离，后台只用开发一次，前台多次开发即可。
  2. 前端页面全用ajax提交，没有表单提交，那么怎么提交数据呢？这时就可以用JSON来表示。
  3. 一句话总结：用JSON传输就可以不用考虑前端用什么语言开发了。

    运行原理

  4. 在服务器获取客户端传来的JSON字符串，并把JSON字符串转换成JAVA对象
  5. 服务器端返回客户端一个java对象转化之后的json格式的字符串

具体原理

  1. 导入对数据类型转换支持的包：jackson-annotations.jar,jackson-core.jar,jackson-databind.jar
  2. 实例化数据类型转换器，那么使用以下注解会自动扫描jakson-annotations包下面的类型转换器，所以不用手动配置bean
  1. 编写Controller
  • 最好使用抽象数据类型而不是简单数据类型
  • @ResponseBody注解是为了将Controller方法返回java对象转换为json响应给客户端
  • @RequestBody注解是为了将读到的内容（json数据）转换为java对象并绑定到Controller方法的参数上
